# PlexLoader - guided-мастер "Скачать сериал целиком"

Проектный план для P2.4: сделать понятный вход в уже существующий
bulk-сценарий загрузки сезонов. Фича не должна заново изобретать подбор сезонов:
она должна провести пользователя к выбору эталонной раздачи, а затем передать
управление текущему механизму `Скачать недостающие сезоны`.

Подробности текущего bulk-подбора сезонов описаны в
[`docs/series-bulk-download.md`](series-bulk-download.md).

## Цель

Пользователь хочет не "найти один сезон", а "скачать сериал целиком". Сейчас это
возможно, но вход неочевиден: нужно сначала выполнить обычный поиск, выбрать
одну сериальную раздачу и только потом нажать `📚 Скачать недостающие сезоны`.

Guided-мастер должен сделать намерение явным:

```text
Хочу скачать сериал целиком
```

и провести пользователя коротким маршрутом:

```text
обычный поиск -> настройки поиска -> "Что скачать: сериал целиком" -> эталонная раздача -> профиль подбора -> план сезонов
```

## Название

Пользовательское название:

```text
📚 Скачать сериал целиком
```

Это название хорошо передает намерение пользователя и понятно без знания Plex или
bulk-плана.

Внутри сценария нужно сразу объяснять, что бот не будет бездумно скачивать все:

```text
Я соберу план по сезонам: пропущу то, что уже есть в Plex или уже качается,
а спорные сезоны покажу отдельно.
```

Так название остается простым, но ожидания становятся честными.

## Нужна ли фича

Да, если PlexLoader хочет быть удобным для пользователя, который приходит с
задачей "хочу сериал", а не с задачей "хочу разобраться в сезонах".

Польза:

- настройка `Что скачать` снижает порог входа без отдельного режима чата;
- пользователь не обязан знать, что bulk-план находится за кнопкой в найденной
  раздаче;
- можно заранее настроить поиск как сериальный и не показывать фильмы;
- текущий bulk-план переиспользуется, поэтому риск меньше, чем у новой логики
  автоскачивания;
- сценарий хорошо ложится в P2.4 Guided-мастера.

Не стоит делать, если цель - "одна команда и бот сам скачал всё без выбора".
Без эталонной раздачи слишком высок риск ошибиться с озвучкой, качеством,
релиз-типом, сезонными паками и дублями.

## MVP

### Вход через настройку "Что скачать"

Отдельную команду в MVP не добавлять: она создает лишний вход и может путать
пользователя. Запуск мастера должен быть не отдельной командой, а вариантом в
обычных настройках поиска:

```text
🎚 Что скачать: одна раздача
```

Значение по умолчанию всегда `Одна раздача`. Это обычный поиск одного релиза:
фильм, сезон, серия, сборник или любая другая раздача.

При нажатии на настройку показывать выбор:

```text
Что скачать?

✅ Одна раздача
📚 Сериал целиком
⬅️ Назад
```

После выбора `📚 Сериал целиком` сценарий остается внутри текущего поискового
диалога. Если пользователь вышел из диалога или вернулся к `Одна раздача`,
следующий текст снова обрабатывается как обычный запрос.

### Ввод запроса

Использовать существующую форму ручного ввода из поиска. Режим
`series_master` включается только после выбора
`Что скачать -> 📚 Сериал целиком`.

Важно:

- бот не должен требовать номер сезона;
- если пользователь случайно ввел `сезон 3`, нормализовать до базового названия
  и показать это в тексте.
- если в запросе явно есть отсылка к сезону, показать мягкую подсказку:

```text
Похоже, вы ищете сезон сериала. Можно найти одну раздачу или собрать план по
всем сезонам.
```

Если сезон в запросе не указан, подсказку не показывать, но настройку
`Что скачать` всё равно оставлять доступной.

### Настройки поиска

Переиспользовать текущие настройки:

- качество;
- оригинальная дорожка;
- субтитры;
- выбранные Jackett-трекеры.

Текст в обычном режиме:

```text
🔍 Поиск

Запрос: «Клиника»

Что скачать: одна раздача
Качество: 1080p
Оригинальная дорожка: выкл
Субтитры: выкл
Трекеры: все
```

Кнопки:

```text
🎚 Что скачать: одна раздача
Качество: 1080p
Оригинальная дорожка: выкл
Субтитры: выкл
Трекеры
🔍 Искать
❌ Отмена
```

Текст в режиме `Сериал целиком`:

```text
📚 Скачать сериал целиком

Запрос: «Клиника»

Настройте поиск эталонной раздачи. Эти параметры потом перейдут в план сезонов.
Предпочитаемую озвучку можно будет выбрать после эталонной раздачи: я покажу
варианты, которые реально нашёл в выбранном релизе.
```

Кнопки:

```text
Качество: 1080p
Оригинальная дорожка: выкл
Субтитры: выкл
Трекеры
🔎 Найти эталон
🎚 Что скачать: сериал целиком
❌ Отмена
```

Повторное нажатие на `🎚 Что скачать: сериал целиком` открывает тот же выбор и
позволяет вернуться к `Одна раздача` без потери запроса и фильтров.

### Озвучки в настройках поиска

Статичный список популярных озвучек в первичный поиск в MVP не добавлять.

Причины:

- список быстро устаревает и будет спорным;
- разные сериалы имеют разные релевантные озвучки;
- длинные сериалы часто меняют набор дорожек по сезонам;
- выбор озвучки уже лучше решается после эталонной раздачи, когда бот видит
  реальные варианты в выбранном релизе;
- статичный список может дать ложное ощущение точности.

Правильный MVP-подход:

1. В первичном поиске оставить только качество, Original, субтитры, трекеры.
2. После выбора эталонной раздачи показать найденные озвучки.
3. Спросить, что важно по озвучке:

```text
Что важно по озвучке?

Из выбранной раздачи:
LostFilm, NewStudio, Original
```

Кнопки:

```text
🔘 Любая из найденных
⚪ Любая русская
⚪ Выбрать конкретную
⚪ Только Original
🔍 Собрать план
⬅️ Назад
❌ Отмена
```

Будущая версия может добавить персональные "любимые озвучки", но это лучше
делать через настройки пользователя или историю, а не через hard-coded список.

### Поиск эталонной раздачи

Переиспользовать `_execute_search` / `_run_search`, но добавить режим:

```text
srch_intent = "series_master"
```

В этом режиме:

- показывать только результаты, которые бот считает сериалами;
- сильнее ранжировать сезонные раздачи и season-pack;
- не показывать фильмы даже при совпадении названия;
- кнопка результата должна вести не в обычный download picker, а в выбор
  эталона для bulk-плана.

Экран результатов:

```text
📚 Эталонная раздача для сериала

Выберите раздачу, по которой я пойму качество, тип релиза и возможные озвучки.
Скачивание начнется только после плана и подтверждения.
```

Кнопки результата:

```text
🎯 1. Выбрать как эталон
🎯 2. Выбрать как эталон
...
⚙️ Настройки
❌ Отмена
```

### После выбора эталона

Не скачивать выбранную раздачу автоматически.

Вместо этого открыть текущий bulk-profile:

```text
📚 Профиль подбора

Эталон:
Клиника / Scrubs / Сезон 3 / 1080p WEB-DL

Из поиска:
1080p
Оригинальная дорожка: нет
Субтитры: нет

Из раздачи:
WEB-DL
Озвучки: LostFilm, NewStudio

Что важно по озвучке?
```

Дальше использовать существующий путь:

```text
profile -> build_series_bulk_plan -> plan screen -> confirm ready -> review disputed
```

## Не-MVP

Не делать в первой версии:

- скачивание всех сезонов полностью автоматически без эталонной раздачи;
- hard-coded список популярных озвучек в первичном поиске;
- отдельный новый planner вместо текущего `series_bulk_planner.py`;
- WebApp-интерфейс;
- персональные предпочтения озвучек;
- автоподбор "лучшей" раздачи без подтверждения пользователя;
- forced-download сезонов, которые уже есть в Plex или активных задачах.

## Технический план

### 1. Entry point

Добавить настройку `🎚 Что скачать` в экран настроек обычного поиска.

Варианты:

- `Одна раздача` - значение по умолчанию;
- `Сериал целиком` - включает `series_master`.

Callback выбора `Сериал целиком` должен:

- проверить доступ;
- установить `context.user_data["srch_intent"] = "series_master"`;
- перерисовать текущий экран настроек в режиме подбора эталона;
- показать подсказку, что озвучка выбирается после эталонной раздачи.

Callback выбора `Одна раздача` должен:

- очистить `context.user_data["srch_intent"]`;
- перерисовать тот же экран настроек как обычный поиск;
- не сбрасывать введённый запрос и выбранные фильтры.

### 2. Conversation state

Лучше не создавать отдельный большой ConversationHandler, а расширить текущий
search conversation:

- вход в режим происходит из текущего `SEARCH_OPTIONS` / `SEARCH_ADVANCED`;
- дальше используются существующие `SEARCH_OPTIONS`, `SEARCH_ADVANCED`,
  `SEARCH_RESULTS`;
- `srch_intent` меняет тексты и поведение кнопок;
- `srch_intent` очищается при `❌ Отмена`, timeout, выборе `Одна раздача` и
  после передачи выбранного эталона в bulk-сценарий.

Это уменьшает риск и не дублирует ручной ввод, настройки качества и выбор
трекеров.

### 3. Search filtering

В `_run_search` после получения результатов, если `srch_intent == "series_master"`:

- оставить результаты, где `_extract_series_base_query(title)` не `None`;
- если результатов нет, показать отдельный empty state:

```text
Не нашёл сериальных раздач

Попробуйте другое название или уберите часть фильтров.
```

Кнопки:

```text
🔄 Попробовать снова
⚙️ Настройки
❌ Отмена
```

### 4. Result action

В режиме `series_master` кнопка результата должна означать:

```text
🎯 Выбрать как эталон
```

а callback должен вести в существующий bulk profile, примерно туда же, куда
сейчас ведёт `srch:bulk_plan:N`.

Важно: не вызывать `_download_and_add` на этом шаге.

### 5. Profile reuse

Переиспользовать текущие поля:

- `srch_series_bulk_source_result`;
- `srch_series_bulk_profile`;
- `srch_series_bulk_results`;
- `srch_picked_quality`;
- выбранные Jackett indexers;
- Original/subtitles from search settings.

Если текущий bulk код ожидает, что пользователь пришел из результата обычного
поиска, добавить тонкую функцию-адаптер:

```python
_series_bulk_prepare_from_reference_result(context, result)
```

Она должна собрать ровно тот же context, который сейчас собирается перед
`bulk_plan`.

### 6. Plan build

Сборка плана не меняется:

- Kinopoisk/fallback определяет сезоны;
- Plex показывает, что уже есть;
- Download Station показывает активные задачи;
- Jackett/Rutracker ищут кандидатов;
- `series_bulk_jobs.json` сохраняет job;
- спорные сезоны разбираются существующим review.

### 7. Тексты и кнопки

Все новые экраны должны соблюдать P0-правила:

- короткий итог сверху;
- `Что дальше` только там, где действительно помогает;
- `❌ Отмена` внутри active conversation;
- `⬅️ Назад` внутри мастера;
- `✖️ Закрыть` только на финальных экранах.

## Тесты

Минимум:

- настройка `🎚 Что скачать` доступна в обычном поиске;
- значение по умолчанию - `Одна раздача`;
- выбор `Сериал целиком` ставит `srch_intent = "series_master"`;
- выбор `Одна раздача` очищает `srch_intent` и сохраняет текущие фильтры;
- подсказка про выбор между одной раздачей и всеми сезонами появляется только
  для запроса с явной отсылкой к сезону;
- настройки поиска переиспользуются;
- search results в `series_master` фильтруют фильмы и оставляют сериалы;
- пустой результат объясняет, что сериальных раздач нет;
- кнопка результата в `series_master` не вызывает `_download_and_add`;
- выбор эталона вызывает bulk-profile/bulk-plan подготовку;
- Original/subtitles/quality/indexers переносятся в bulk profile;
- cancel очищает `srch_intent` вместе с поисковым контекстом.

## Риски

### Пользователь ожидает "скачать всё сразу"

Название `Скачать сериал целиком` может звучать как мгновенное скачивание.
Компенсация: в тексте режима явно написать, что бот сначала соберёт план и
попросит подтверждение.

### Неверный эталон

Если пользователь выбрал не тот сезон, не ту озвучку или странный pack, bulk
план может быть хуже. Компенсация: профиль подбора должен показывать, что бот
извлёк из эталона, и давать `⬅️ Назад`.

### Слишком широкий поиск

По названию сериала могут прийти фильмы, спецвыпуски, похожие сериалы. В
`series_master` нужен фильтр "только сериалы" и хороший empty state.

### Озвучки не распознаны

Если бот не понял озвучки в эталоне, не надо притворяться. Показать:

```text
Озвучки: не распознал
```

и оставить безопасные политики:

```text
Любая русская
Только Original
```

## Пошаговая реализация

1. Документ и UX-решения.
2. Настройка `🎚 Что скачать` в настройках поиска.
3. `srch_intent = "series_master"` в search context и возврат к
   `Одна раздача`.
4. Фильтр результатов "только сериалы".
5. Отдельные тексты и кнопки результатов для выбора эталона.
6. Переход из эталона в текущий bulk profile.
7. Тесты переноса настроек поиска в bulk profile.
8. README: добавить пользовательский сценарий без отдельной команды.
9. `docs/bot-structure.md`: обновить callbacks и search intent.

## Definition of done

- Пользователь видит настройку `Что скачать: одна раздача` в обычном поиске.
- Пользователь может выбрать `Сериал целиком` из этой настройки.
- Пользователь может вернуться к `Одна раздача` без потери запроса и фильтров.
- Бот использует обычный запрос и обычные настройки поиска.
- Результаты поиска ограничены сериальными раздачами.
- Пользователь выбирает эталонную раздачу.
- После выбора эталона открывается текущий bulk profile.
- Скачивание не стартует до плана и подтверждения.
- Настройки качества, Original, субтитров и трекеров переносятся в bulk.
- Спорные сезоны остаются в review, а не скачиваются автоматически.
- Тесты покрывают вход, фильтр, выбор эталона и перенос настроек.
