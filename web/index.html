PlexLoader

Telegram-бот для домашнего медиасервера на Synology + Plex: ищет раздачи, добавляет их в Download Station, следит за загрузкой и сообщает, когда контент появился в Plex.

Поиск

Пишите название фильма или сериала, отправляйте magnet, torrent-файл, ссылку Кинопоиска или голосовое сообщение.

Plex

Бот проверяет дубли, ждёт появления контента в библиотеке и даёт кнопку открытия в Plex.

Сериалы

Можно следить за новыми сериями, ждать финал сезона или собрать план скачивания недостающих сезонов.

Перед установкой

PlexLoader ставится в Docker на Synology. Базовый мастер настраивает Telegram-бота и Synology Download Station. Rutracker, Jackett, Plex, /new и OpenAI подключаются после первого запуска через .env.

На Synology

  1. Откройте DSM → Package Center и установите Container Manager.
  2. Убедитесь, что установлен и запущен Download Station.
  3. Включите SSH: DSM → Панель управления → Терминал и SNMP → Включить SSH.

Доступы

  1. Создайте Telegram-бота в @BotFather.
  2. Подготовьте DSM-пользователя для PlexLoader, например tg_bot_ds.
  3. Дайте этому пользователю доступ к Download Station и папке загрузки, например video.

1. Получить Telegram token

  1. Откройте @BotFather в Telegram.
  2. Отправьте команду /newbot.
  3. Введите имя бота, например PlexLoader Home.
  4. Введите username, который заканчивается на bot, например my_plexloader_bot.
  5. Скопируйте token вида 123456789:AA.... Его попросит мастер установки.

Token нельзя публиковать. Если token утёк, в BotFather используйте /revoke.

2. Подключиться к NAS

На компьютере откройте терминал и подключитесь к Synology по SSH:

ssh your_user@192.168.1.10

Замените your_user и IP-адрес на свои. Пользователь должен иметь право запускать Docker.

3. Запустить установщик

В SSH-сессии выполните:

curl -fsSL https://raw.githubusercontent.com/KiMorev/tg-torrent-bot/main/install.sh | sh

Если на NAS нет curl, используйте:

wget -qO- https://raw.githubusercontent.com/KiMorev/tg-torrent-bot/main/install.sh | sh

Установщик создаст папку /volume1/docker/plexloader, скачает compose.yaml и мастер настройки, создаст .env, запустит контейнер и покажет логи при ошибке.

4. Что спросит мастер

  1. BOT_TOKEN. Вставьте token из BotFather.
  2. Telegram chat_id. Мастер попросит написать /start вашему боту и попробует сам найти chat_id. Ручной ввод нужен только если Telegram не отдаст update.
  3. DSM URL. Обычно можно оставить https://host.docker.internal:5001. Это адрес DSM из контейнера.
  4. DSM account/password. Введите пользователя Synology, которого подготовили для PlexLoader.
  5. Папка назначения. Обычно video или другая папка, настроенная в Download Station.

Если DSM использует самоподписанный сертификат, мастер сам попробует безопасный fallback DS_VERIFY_SSL=false. Если Download Station не проходит проверку, мастер покажет причину и предложит ввести данные заново.

5. Проверить запуск

После завершения установки откройте своего Telegram-бота:

  • /ping должен ответить pong.
  • /status должен показать задачи Download Station.
  • /admin должен открыть панель диагностики.
  • /help покажет доступные функции с учётом включённых интеграций.

Если бот не отвечает, посмотрите логи на NAS:

cd /volume1/docker/plexloader
docker compose logs --tail=100 tg_torrent_drop

Что подключить после базовой установки

Откройте файл /volume1/docker/plexloader/.env и добавьте только те интеграции, которые нужны. После изменения .env примените настройки командой docker compose up -d из папки установки.

Rutracker

Нужны логин и пароль аккаунта Rutracker.

RUTRACKER_USERNAME=login
RUTRACKER_PASSWORD=password

Jackett

В Jackett откройте главную страницу и скопируйте API Key. URL обычно выглядит как http://host:9117.

JACKETT_URL=http://192.168.1.10:9117
JACKETT_API_KEY=replace_me

Plex

Укажите URL Plex Media Server и token. Без Plex бот всё равно скачивает, но не проверяет дубли и не присылает кнопку просмотра.

PLEX_URL=http://192.168.1.10:32400
PLEX_TOKEN=replace_me

OpenAI

Нужен API key из личного кабинета OpenAI. Он включает голосовой поиск и GPT-подсказки.

OPENAI_API_KEY=sk-...

Подробный список переменных и сценариев — в README на GitHub.

Plex redirect для телефона

Файл plex.html нужен для кнопки Смотреть в Plex. Если бот откроет его с параметрами key и server, страница попробует открыть приложение Plex и оставит fallback в Plex Web.

Если сайт plex.morplex.ru уже смотрит в папку /web/plex-redirect, положите файлы так:

/web/plex-redirect/plex.html
/web/plex-redirect/index.html

В .env укажите базовый адрес без параметров:

PLEX_DEEPLINK_BASE_URL=https://plex.morplex.ru/plex.html

Параметры key и server бот добавляет сам, когда Plex нашёл конкретный фильм или сезон.

Обновление

Если используется стандартный compose.yaml, Watchtower проверяет новый образ и обновляет PlexLoader автоматически. Для ручного обновления:

cd /volume1/docker/plexloader
docker compose pull
docker compose up -d

Если менялись только файлы index.html или plex.html на Web Station, контейнер бота перезапускать не нужно.

Не размещайте на публичной странице токены, пароли, .env, chat_id, внутренние адреса NAS или Plex token. Эта страница содержит только общую инструкцию.
