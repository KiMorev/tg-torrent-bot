PlexLoader

Telegram-бот для домашнего медиасервера на Synology + Plex: ищет раздачи, добавляет их в Download Station, следит за загрузкой и сообщает, когда контент появился в Plex.

Поиск

Пишите название фильма или сериала, отправляйте magnet, torrent-файл, ссылку Кинопоиска или голосовое сообщение.

Plex

Бот проверяет дубли, ждёт появления контента в библиотеке и даёт кнопку открытия в Plex.

Сериалы

Можно следить за новыми сериями, ждать финал сезона или собрать план скачивания недостающих сезонов.

Установка одной командой

Подключитесь к Synology по SSH пользователем, который может запускать Docker, и выполните:

curl -fsSL https://raw.githubusercontent.com/KiMorev/tg-torrent-bot/main/install.sh | sh

Если на NAS нет curl, используйте wget -qO- https://raw.githubusercontent.com/KiMorev/tg-torrent-bot/main/install.sh | sh.

Что подготовить

  1. Synology с установленным Container Manager или Docker.
  2. Telegram-бот из @BotFather: нужен только token.
  3. DSM-пользователь для PlexLoader с доступом к Download Station и папке загрузки.

Rutracker, Jackett, Plex, подборка /new и OpenAI подключаются после базовой установки через .env.

Что сделает установщик

  • Проверит Docker и Docker Compose.
  • Создаст папку /volume1/docker/plexloader.
  • Скачает compose.yaml и мастер настройки.
  • Проведёт по Telegram token, chat_id, DSM URL, логину, паролю и папке назначения.
  • Создаст .env, запустит контейнер и покажет последние логи, если запуск не удался.

Проверка после запуска

  • /ping должен ответить pong.
  • /status должен показать задачи Download Station.
  • /admin должен открыть панель диагностики.
  • /help покажет доступные функции с учётом включённых интеграций.

Plex redirect

Файл plex.html нужен для кнопки Смотреть в Plex на телефоне. Если открыть его без параметров, он покажет служебный экран. Если бот откроет его с параметрами key и server, страница попробует открыть приложение Plex и оставит fallback в Plex Web.

PLEX_DEEPLINK_BASE_URL=https://your-domain.example/plex.html

Обновление

Если включён Watchtower из стандартного compose.yaml, PlexLoader обновляется автоматически после успешного CI и публикации нового Docker-образа.

Для ручного обновления из папки установки:

docker compose pull
docker compose up -d

Не размещайте на публичной странице токены, пароли, .env, chat_id, внутренние адреса NAS или Plex token. Подробная документация: README на GitHub.
